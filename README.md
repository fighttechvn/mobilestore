# Mobile Store Listing Preview

A single-file, JSON-driven mockup of how an app looks on the **Apple App Store**
and **Google Play** store pages — useful for reviewing listing copy, screenshots
and graphics before you submit.

🔗 **Live demo:** https://fighttechvn.github.io/mobilestore/

> This deployment uses **dummy demo data** (a fictional "PulseFit" app). No real
> app credentials or identifiers are included.

## Features

- **Two store layouts** — toggle between App Store and Google Play renderings.
- **Localized** — switch locales; per-locale copy from one JSON file.
- **ASO checks** — live character-count limits per store field (name, subtitle,
  keywords, descriptions, changelog…).
- **Edit JSON** — edit the listing inline and hit *Apply & preview*; download the
  result as `listing.json`.
- **Load JSON…** — open any `listing.json` to preview your own app.
- **🖼️ Drag & drop images** — drop image files **or a whole folder** anywhere on
  the page (or use *Load images…*). They're auto-sorted into iPhone / iPad /
  phone / tablet galleries, app icon and feature graphic, by filename hints and
  aspect ratio. Phone shots mirror into both stores; tablet shots likewise.
- **✅ Release checklist** — click **Release checklist** in the top bar for an
  interactive, in-browser checklist of every step to submit to both stores
  (progress is saved locally). Same content as the [Release checklist](#release-checklist)
  below.

## Use it with your own app

1. Open the live demo (or run locally — see below).
2. Click **Load JSON…** and pick your `listing.json` (same shape as the bundled
   `listing.json`).
3. Drag your screenshots (or their folder) onto the page to fill the galleries.
4. Tweak copy in **Edit JSON**, watch the **ASO checks**, then **Download
   listing.json**.

## Run locally

```bash
python3 -m http.server 8092
# open http://localhost:8092/
```

(Browsers block `fetch()` over `file://`, so it must be served over http.)

## `listing.json` shape

```jsonc
{
  "app":   { "name", "androidPackage", "iosBundleId", "versionName", "icon", "developer", "rating", "downloads", "locales": [...], ... },
  "limits": { "appstore": {...}, "googleplay": {...} },     // ASO character limits
  "locales": { "en-US": { "displayName", "appstore": {...}, "googleplay": {...}, "fullDescription", "whatsNew" }, ... },
  "screenshots": { "iphone": [{file,label,size}], "ipad": [...], "phone": [...], "tablet": [...] },
  "graphics": { "appIcon": {status,file,spec}, "featureGraphic": {status,file,spec} }
}
```

Generated by the `store-release` skill.

## Release checklist

A first App Store + Google Play submission, in order. Tags: **`[API]`** scriptable
(fastlane/spaceship) · **`[WEB UI]`** App Store Connect / Play Console only ·
**`[DECLARATION]`** legal/business answer — don't fabricate, ask the app owner.
The same checklist is built into the site (top bar → **Release checklist**).

### Accounts & signing
- [ ] Apple Developer Program active; team id noted
- [ ] Google Play Developer account active
- [ ] `[API]` App Store Connect API key created; `.p8` saved to `~/.appstoreconnect/private_keys/`
- [ ] `[API]` Google Play service-account JSON (role: Release manager) in `.env.prod`
- [ ] iOS distribution cert + App Store provisioning profile (or Xcode automatic)
- [ ] Android keystore available; Play App Signing enabled
- [ ] All secrets gitignored — nothing committed to the app repo

### App records (one-time)
- [ ] `[WEB UI]` App Store Connect app record created (API key cannot — 403)
- [ ] `[WEB UI]` Play Console app created
- [ ] Bundle id / applicationId final (permanent once published)

### Listing content
- [ ] `[API]` iOS metadata per locale in `ios/fastlane/metadata/<locale>/`
- [ ] `[API]` Android metadata per locale in `android/fastlane/metadata/android/<locale>/`
- [ ] iOS screenshots: iPhone 6.9"/6.7" (`APP_IPHONE_67`) + iPad 12.9" (`APP_IPAD_PRO_3GEN_129`)
- [ ] Android graphics: icon, feature graphic, phone + 7"/10" tablet shots
- [ ] Copy within each store's character limits (use the ASO checks)
- [ ] Re-verify screenshots aren't flagged "missing" (assets can show a count yet fail to finalize)

### Build
- [ ] `version:` bumped in `pubspec.yaml`
- [ ] `[API]` iOS binary built + uploaded (no-codesign archive → `exportArchive` → `altool`)
- [ ] iOS build reaches state **VALID** (processing 5–30 min — poll before attaching)
- [ ] `[API]` iOS build attached to the version; version string == `CFBundleShortVersionString`
- [ ] `[API]` Android AAB uploaded to a track (`fastlane internal`)

### App Store version requirements (the "Submit" blockers)
- [ ] `[API]` Copyright set on the version
- [ ] `[API]` `[DECLARATION]` Content Rights declaration set
- [ ] `[API]` `[DECLARATION]` Age Rating questionnaire — all descriptors None → 4+ (also requires `ageAssurance`)
- [ ] `[WEB UI]` `[DECLARATION]` App Privacy data-collection answers **published**
- [ ] `[WEB UI]` Pricing set (e.g. Free)
- [ ] Export compliance answered (`ITSAppUsesNonExemptEncryption` in Info.plist)

### Submit
- [ ] `[API]` iOS: create review submission → add version → `submit_for_review`
- [ ] iOS: version state moves to `WAITING_FOR_REVIEW`
- [ ] `[WEB UI]` Android: every Play Dashboard task green (data safety, content rating, audience, privacy)
- [ ] `[API]` Android: promote tested internal build to production

### Known first-submission rejections
- [ ] **90771** — `UIBackgroundModes` without a matching capability → remove unused mode
- [ ] **ITMS-90683** — missing usage string (e.g. `NSPhotoLibraryUsageDescription`) → add it
- [ ] **"No data"** at `fetch_app_store_review_detail` → create review detail via spaceship
- [ ] **invalid byte sequence in US-ASCII** → `export LANG=en_US.UTF-8 LC_ALL=en_US.UTF-8`
- [ ] **fastlane/`gh_inspector` missing** → use Homebrew ruby + `bundle exec`
